## Story
**As a** super admin,
**I want** the admin web panel to have navigation and auth,
**so that** I can access the management dashboard.

## Acceptance Criteria
1. Next.js App Router com layout de sidebar navigation (dark theme, 256px width)
2. Sidebar com links: Dashboard, Restaurantes, Usuarios, Assinaturas, Notificacoes, Cidades
3. Paginas placeholder para cada secao do sidebar
4. Auth middleware que redireciona para login se nao autenticado ou `role != super_admin`
5. Tela de login funcional com email/senha
6. Supabase client configurado para SSR (server.ts) e client-side (client.ts)
7. Layout responsivo basico: sidebar persistente em desktop, drawer em mobile
8. `packages/ui` com componentes basicos: Button, Input, Card, Table (usados por ambos os admin webs)

## Dev Notes

### Auth Middleware (da arquitetura secao 9.5)

```typescript
// apps/admin-web/middleware.ts
import { createServerClient } from '@supabase/ssr'
import { NextResponse } from 'next/server'
import type { NextRequest } from 'next/server'

export async function middleware(request: NextRequest) {
  const response = NextResponse.next()

  const supabase = createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    { cookies: { getAll: () => request.cookies.getAll() } }
  )

  const { data: { user } } = await supabase.auth.getUser()

  // Redirect to login if not authenticated
  if (!user && !request.nextUrl.pathname.startsWith('/login')) {
    return NextResponse.redirect(new URL('/login', request.url))
  }

  if (user) {
    // admin-web requires super_admin
    const { data: profile } = await supabase
      .from('profiles')
      .select('role')
      .eq('id', user.id)
      .single()

    if (profile?.role !== 'super_admin') {
      return NextResponse.redirect(new URL('/login?error=unauthorized', request.url))
    }
  }

  return response
}

export const config = {
  matcher: ['/((?!_next/static|_next/image|favicon.ico|login).*)']
}
```

### Supabase Clients (da arquitetura secao 9.4)

```typescript
// apps/admin-web/lib/supabase/client.ts
import { createBrowserClient } from '@supabase/ssr'
import { Database } from '@maisum/shared/types/supabase'

export const createClient = () =>
  createBrowserClient<Database>(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!
  )

// apps/admin-web/lib/supabase/server.ts
import { createServerClient } from '@supabase/ssr'
import { cookies } from 'next/headers'
import { Database } from '@maisum/shared/types/supabase'

export const createClient = () => {
  const cookieStore = cookies()
  return createServerClient<Database>(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    { cookies: { getAll: () => cookieStore.getAll() } }
  )
}
```

### packages/ui Components

Os componentes de `packages/ui` sao compartilhados entre admin-web e restaurant-web. Devem usar Tailwind com o shared preset e ser compatíveis com shadcn/ui + Radix patterns:

**Button specs (da frontend spec):**
- Variants: primary (#FF6B35 bg, white text), secondary (transparent bg, primary border+text), ghost, danger, success
- Sizes: sm (32px h, 12px font), md (44px h, 14px font), lg (52px h, 16px font)
- States: default → hover (opacity 0.9) → pressed (scale 0.97) → disabled (opacity 0.5) → loading (spinner)

**Input specs:**
- Height: 48px, padding 16px, radius 6px (sm)
- States: default (neutral-300 border), focus (primary border), error (red border), disabled
- Floating label pattern

**Card specs:**
- Padding: 16px (space-4), radius 16px (lg), shadow-sm
- Hover: shadow-md

**Table specs:**
- Header: bg neutral-100, font caption semibold
- Rows: alternating white/neutral-50
- Hover: neutral-100
- Pagination: previous/next + page numbers
