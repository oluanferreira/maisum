## Story
**As a** restaurant admin,
**I want** to respond to customer messages,
**so that** I can provide good customer service.

## Acceptance Criteria
1. Pagina "Chat" no painel do restaurante lista todas as conversas, ordenadas por ultima mensagem (FR26, FR52)
2. Badge de nao lidas no sidebar (menu lateral) e em cada conversa na listagem (FR26)
3. Tela de conversa com: header (nome do cliente + avatar), mensagens em bolhas, input de texto + botao enviar (FR26)
4. Mensagens em tempo real — Supabase Realtime subscription ativa, sem necessidade de refresh (FR26, NFR7)
5. Indicador de "nova mensagem" com som e notificacao no browser quando nova mensagem chega (FR28)
6. Filtro de conversas: todas, nao respondidas, resolvidas (FR52)

**Split-view Layout Pattern:**
```typescript
// Layout responsivo: sidebar de conversas + area de mensagens
// Mobile: lista de conversas → tap abre conversa (stack navigation)
// Desktop/Tablet: split-view lado a lado

<div className="flex h-[calc(100vh-4rem)]">
  {/* Lista de conversas (1/3 da tela) */}
  <aside className="w-1/3 border-r overflow-y-auto">
    <ConversationList />
  </aside>
  {/* Area de mensagens (2/3 da tela) */}
  <main className="flex-1 flex flex-col">
    <ConversationMessages />
  </main>
</div>
```


**Browser Notification API:**
```typescript
// Solicitar permissao
const requestNotificationPermission = async () => {
  if ('Notification' in window && Notification.permission === 'default') {
    await Notification.requestPermission()
  }
}

// Enviar notificacao
const showNotification = (title: string, body: string) => {
  if (Notification.permission === 'granted') {
    new Notification(title, { body, icon: '/logo.png' })
  }
}

// Som de notificacao
const notificationSound = new Audio('/sounds/notification.mp3')
notificationSound.play()
```

**Filtro de Conversas com Supabase:**
```typescript
// "Nao respondidas" — ultima mensagem e do usuario
const { data } = await supabase
  .from('conversations')
  .select('*, messages!inner(sender_role)')
  .eq('restaurant_id', myRestaurantId)
  .order('last_message_at', { ascending: false })

// Filtrar client-side ou usar RPC para performance
```

**Key decisions:**
- Split-view no desktop para UX profissional (similar Intercom/Zendesk)
- Browser Notification API para alertar mesmo quando tab nao esta em foco
- Som de notificacao discreto (nao intrusivo) — arquivo MP3 curto
- Filtro "nao respondidas" baseado em sender_role da ultima mensagem
- Subscription global para notificar novas mensagens em qualquer conversa
