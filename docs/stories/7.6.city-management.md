## Dev Notes

**Cities Table DDL (from Architecture doc section 5.1):**
```sql
CREATE TABLE cities (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  name TEXT NOT NULL,
  state TEXT NOT NULL,
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMPTZ DEFAULT now()
);
```

**RLS Policies (from Architecture doc):**
```sql
CREATE POLICY "Anyone can view active cities" ON cities FOR SELECT USING (is_active = true);
CREATE POLICY "Super admin full access to cities" ON cities FOR ALL USING (get_user_role() = 'super_admin');
```

**Restaurants query impact:**
- A RLS policy de restaurants ja filtra `WHERE is_active = true`, mas NAO verifica `cities.is_active`
- Necessario: adicionar check na query de discovery do mobile, ou ajustar RLS de restaurants para verificar cidade ativa:
```sql
-- Opcao 1: Ajustar query no frontend (mais simples)
SELECT r.* FROM restaurants r
JOIN cities c ON r.city_id = c.id
WHERE r.is_active = true AND c.is_active = true;

-- Opcao 2: Ajustar RLS (mais seguro, mas complexo)
-- Avaliar impacto de performance
```
