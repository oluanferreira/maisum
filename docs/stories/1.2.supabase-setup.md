7. RLS habilitado em todas as tabelas com policies basicas (users veem seus dados, super_admin ve tudo, cities publicas)
8. Helper functions `get_user_role()` e `get_my_restaurant_id()` criadas como SECURITY DEFINER STABLE
9. Trigger `on_auth_user_created` cria profile automaticamente com role 'user'
10. Seed com cidade "Jequie, BA" ativa
11. `supabase start` roda localmente sem erros
12. `packages/shared` exporta types gerados do Supabase via `supabase gen types typescript --local`

## Dev Notes

### ENUM Types (completo da arquitetura)

```sql
CREATE TYPE user_role AS ENUM ('user', 'restaurant_admin', 'super_admin');
CREATE TYPE plan_type AS ENUM ('monthly', 'annual');
CREATE TYPE subscription_status AS ENUM ('active', 'cancelled', 'past_due', 'expired');
CREATE TYPE coupon_status AS ENUM ('available', 'used', 'expired');
CREATE TYPE coupon_source AS ENUM ('subscription', 'referral', 'review', 'social');
CREATE TYPE benefit_category AS ENUM ('prato', 'drink', 'sobremesa', 'combo');
CREATE TYPE referral_status AS ENUM ('pending', 'completed');
CREATE TYPE social_proof_status AS ENUM ('pending', 'approved', 'rejected');
CREATE TYPE social_proof_type AS ENUM ('screenshot', 'link');
CREATE TYPE payment_status AS ENUM ('pending', 'paid', 'failed', 'refunded');
CREATE TYPE payment_method AS ENUM ('pix', 'credit_card', 'boleto');
```

### Tabelas DDL (referencia completa)

Todas as tabelas seguem o DDL definido na secao 5.1 do `docs/fullstack-architecture.md`. Pontos-chave:

- **profiles**: `id UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE` — nao tem uuid_generate_v4(), usa o ID do auth.users
- **profiles.referral_code**: `TEXT UNIQUE DEFAULT encode(gen_random_bytes(6), 'hex')` — codigo de indicacao auto-gerado
- **profiles.extra_coupons_this_month**: `INT DEFAULT 0` — contador de cupons extras (reseta mensalmente)
- **restaurants.admin_user_id**: `UUID REFERENCES profiles(id)` — vincula restaurante ao admin
- **coupons.hmac_signature**: Usa `hmac()` com `current_setting('app.hmac_secret', true)`
- **restaurants.photos**: `TEXT[] DEFAULT '{}'` — array PostgreSQL
- **conversations**: `UNIQUE(user_id, restaurant_id)` — 1 conversa por par
- **reviews**: `UNIQUE(user_id, coupon_id)` — 1 review por cupom usado

### Helper Functions (RLS)

```sql
-- Retorna role do usuario autenticado
CREATE OR REPLACE FUNCTION get_user_role()
RETURNS user_role AS $$
  SELECT role FROM profiles WHERE id = auth.uid();
$$ LANGUAGE sql SECURITY DEFINER STABLE;

-- Retorna restaurant_id do admin autenticado
CREATE OR REPLACE FUNCTION get_my_restaurant_id()
RETURNS UUID AS $$
  SELECT id FROM restaurants WHERE admin_user_id = auth.uid() LIMIT 1;
$$ LANGUAGE sql SECURITY DEFINER STABLE;
```

### Trigger handle_new_user

```sql
CREATE OR REPLACE FUNCTION handle_new_user()
RETURNS TRIGGER AS $$
BEGIN
  INSERT INTO profiles (id, full_name, role)
  VALUES (
    NEW.id,
    COALESCE(NEW.raw_user_meta_data->>'full_name', ''),
    'user'
  );
  RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

CREATE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users
  FOR EACH ROW EXECUTE FUNCTION handle_new_user();
```
