## Story
**As a** user,
**I want** to receive push notifications for important events,
**so that** I don't miss anything relevant.

## Acceptance Criteria
1. Expo Notifications configurado para iOS e Android com permissoes e certificados (FR55, FR56)
2. Tabela `push_tokens` criada: id (UUID PK), user_id (FK profiles), token (TEXT NOT NULL), platform (TEXT CHECK 'ios'|'android'), created_at (TIMESTAMPTZ), UNIQUE(user_id, token) (FR55)
3. Token registrado automaticamente ao fazer login no app, atualizado quando muda (token refresh) (FR55)
4. Edge Function `send-push` envia notificacao via Expo Push API (`POST https://exp.host/--/api/v2/push/send`) (FR55, FR56)
5. Notificacoes para cliente: nova mensagem no chat, cupom extra liberado (review/referral/social), cupom expirando (24h antes), novo restaurante parceiro na cidade (FR55)
6. Notificacoes para restaurante: novo cupom para validar, nova mensagem no chat, nova avaliacao recebida (FR56)
7. Deep linking: tap na notificacao abre a tela relevante no app (conversa, cupons, restaurante, etc.) (FR55)
8. Painel admin +um: tela para enviar push notification segmentado (todos usuarios, usuarios de uma cidade, todos restaurantes) (FR47)

## Dev Notes

**push_tokens Table DDL (from Architecture section 5.1):**
```sql
CREATE TABLE push_tokens (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  token TEXT NOT NULL,
  platform TEXT NOT NULL CHECK (platform IN ('ios', 'android')),
  created_at TIMESTAMPTZ DEFAULT now(),
  UNIQUE(user_id, token)
);

-- RLS
ALTER TABLE push_tokens ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users manage own tokens" ON push_tokens FOR ALL USING (user_id = auth.uid());
```

**Expo Push API (from Architecture section 7.3):**
```typescript
// supabase/functions/send-push/index.ts
import { serve } from 'https://deno.land/std/http/server.ts'
import { createClient } from 'https://esm.sh/@supabase/supabase-js'

serve(async (req: Request) => {
  const { user_ids, title, body, data } = await req.json()

  const supabase = createClient(
    Deno.env.get('SUPABASE_URL')!,
    Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!
  )

  // Buscar tokens dos usuarios
  const { data: tokens } = await supabase
    .from('push_tokens')
    .select('token')
    .in('user_id', user_ids)

  if (!tokens?.length) {
    return new Response(JSON.stringify({ sent: 0 }), { status: 200 })
  }

  // Enviar via Expo Push API (batch de ate 100)
  const messages = tokens.map(t => ({
    to: t.token,
    sound: 'default',
    title,
    body,
    data,
  }))

  const chunks = chunkArray(messages, 100)
  let totalSent = 0

  for (const chunk of chunks) {
    const response = await fetch('https://exp.host/--/api/v2/push/send', {
      method: 'POST',
      headers: {
        'Accept': 'application/json',
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${Deno.env.get('EXPO_PUSH_ACCESS_TOKEN')}`,
      },
      body: JSON.stringify(chunk),
    })

    const result = await response.json()
    totalSent += chunk.length

    // Tratar tokens invalidos
    result.data?.forEach((ticket: any, index: number) => {
      if (ticket.status === 'error' && ticket.details?.error === 'DeviceNotRegistered') {
        // Remover token invalido
        supabase.from('push_tokens').delete().eq('token', chunk[index].to)
      }
    })
  }

  return new Response(JSON.stringify({ sent: totalSent }), { status: 200 })
})

function chunkArray<T>(arr: T[], size: number): T[][] {
  const chunks: T[][] = []
  for (let i = 0; i < arr.length; i += size) {
    chunks.push(arr.slice(i, i + size))
  }
  return chunks
}
```

**Deep Linking com Expo Router:**
```typescript
// app/_layout.tsx
import * as Notifications from 'expo-notifications'
import { router } from 'expo-router'

// Foreground handler
Notifications.setNotificationHandler({
  handleNotification: async () => ({
    shouldShowAlert: true,
    shouldPlaySound: true,
    shouldSetBadge: true,
  }),
})

// Tap handler (deep link)
const lastNotificationResponse = Notifications.useLastNotificationResponse()
useEffect(() => {
  if (lastNotificationResponse) {
    const data = lastNotificationResponse.notification.request.content.data
    switch (data.type) {
      case 'chat':
        router.push(`/(tabs)/chat/${data.conversationId}`)
        break
      case 'coupon_extra':
      case 'coupon_expiring':
        router.push('/(tabs)/coupons')
        break
      case 'new_restaurant':
        router.push(`/restaurant/${data.restaurantId}`)
        break
    }
  }
}, [lastNotificationResponse])
```

**Key decisions:**
- Expo Push Token (nao FCM/APNs direto) — simplifica para cross-platform
- UPSERT de token para lidar com token refresh automaticamente
- Edge Function send-push como servico reutilizavel (chamada por outros triggers)
- Batch sending (100 por request) para respeitar limites do Expo
- Remocao automatica de tokens invalidos (DeviceNotRegistered)
- Deep linking via Expo Router (scheme: maisum://)
- Cron job diario para notificacao de cupom expirando (nao real-time)
