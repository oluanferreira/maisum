## Story
**As a** client user,
**I want** to generate a QR code for my coupon,
**so that** the restaurant can validate my "+um" benefit.

## Acceptance Criteria
1. Na tela de detalhes do restaurante, botao "Usar Cupom" gera QR Code (FR16)
2. QR Code contem payload: coupon_id (UUID) + user_id + restaurant_id + timestamp + HMAC-SHA256 signature (FR17)
3. Tela dedicada de QR Code: alto contraste (fundo branco puro, QR preto), brilho da tela maximizado automaticamente (NFR2)
4. Timer visivel mostrando tempo restante (15 minutos de validade) (FR18)
5. Botao "Regenerar" para criar novo QR Code se expirou (FR18)
6. QR Code pre-gerado e cacheado localmente com AsyncStorage (funciona offline) (NFR2)
7. Verificacao: usuario tem cupom disponivel E nao tem cupom ativo neste restaurante (FR15)
8. Se nao tem cupom disponivel, mostra mensagem explicativa (FR14)

## Dev Notes

**QR Payload Generation (from Architecture doc section 6.3):**
```typescript
// apps/mobile/utils/qr-code.ts
import { hmacSHA256 } from './hmac'

interface QRPayload {
  coupon_id: string
  user_id: string
  restaurant_id: string
  timestamp: number
  signature: string
}

function generateQRPayload(
  couponId: string,
  userId: string,
  restaurantId: string,
  secret: string
): QRPayload {
  const timestamp = Date.now()
  const data = `${couponId}:${userId}:${restaurantId}:${timestamp}`
  const signature = hmacSHA256(data, secret)

  return {
    coupon_id: couponId,
    user_id: userId,
    restaurant_id: restaurantId,
    timestamp,
    signature,
  }
}
```

**AsyncStorage caching (offline):**
```typescript
// Pre-generate and cache
const payload = generateQRPayload(couponId, userId, restaurantId, secret)
await AsyncStorage.setItem(
  `qr:${couponId}:${restaurantId}`,
  JSON.stringify(payload)
)

// Load from cache (offline-first)
const cached = await AsyncStorage.getItem(`qr:${couponId}:${restaurantId}`)
if (cached) {
  const payload = JSON.parse(cached) as QRPayload
  // Check if still valid (< 15min)
  if (Date.now() - payload.timestamp < 15 * 60 * 1000) {
    return payload
  }
}
```

**HMAC Secret management:**
- Secret armazenado em `EXPO_PUBLIC_HMAC_SECRET` (env var)
- Em producao: rotacionar periodicamente
- Server-side usa mesmo secret via `HMAC_SECRET` nas Edge Functions
- A validacao final e SEMPRE server-side (RPC `validate_coupon`)
