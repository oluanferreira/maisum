## Story
**As a** restaurant admin,
**I want** the restaurant panel to have navigation and auth,
**so that** I can access my restaurant's management tools.

## Acceptance Criteria
1. Next.js App Router com layout de sidebar navigation (mesmos specs visuais do admin-web: dark theme neutral-900, 256px)
2. Sidebar com links: Dashboard, Beneficios, Validar Cupom, Chat, Avaliacoes, Perfil
3. Paginas placeholder para cada secao do sidebar
4. Auth middleware que redireciona para login se nao autenticado ou `role != restaurant_admin`
5. Tela de login funcional com email/senha
6. Supabase client configurado para SSR (server.ts) e client-side (client.ts) — mesmo pattern do admin-web
7. Layout responsivo com prioridade tablet (uso no balcao): botoes 48px+, touch targets generosos
8. Reutiliza componentes de `packages/ui` (Button, Input, Card, Table)

## Dev Notes

### Middleware (adaptado da arquitetura secao 9.5 para restaurant_admin)

```typescript
// apps/restaurant-web/middleware.ts
import { createServerClient } from '@supabase/ssr'
import { NextResponse } from 'next/server'
import type { NextRequest } from 'next/server'

export async function middleware(request: NextRequest) {
  const response = NextResponse.next()

  const supabase = createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    { cookies: { getAll: () => request.cookies.getAll() } }
  )

  const { data: { user } } = await supabase.auth.getUser()

  // Redirect to login if not authenticated
  if (!user && !request.nextUrl.pathname.startsWith('/login')) {
    return NextResponse.redirect(new URL('/login', request.url))
  }

  if (user) {
    // restaurant-web requires restaurant_admin
    const { data: profile } = await supabase
      .from('profiles')
      .select('role')
      .eq('id', user.id)
      .single()

    if (profile?.role !== 'restaurant_admin') {
      return NextResponse.redirect(new URL('/login?error=unauthorized', request.url))
    }
  }

  return response
}

export const config = {
  matcher: ['/((?!_next/static|_next/image|favicon.ico|login).*)']
}
```

### Supabase Clients (mesmo pattern do admin-web)

```typescript
// apps/restaurant-web/lib/supabase/client.ts
import { createBrowserClient } from '@supabase/ssr'
import { Database } from '@maisum/shared/types/supabase'

export const createClient = () =>
  createBrowserClient<Database>(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!
  )

// apps/restaurant-web/lib/supabase/server.ts
import { createServerClient } from '@supabase/ssr'
import { cookies } from 'next/headers'
import { Database } from '@maisum/shared/types/supabase'

export const createClient = () => {
  const cookieStore = cookies()
  return createServerClient<Database>(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    { cookies: { getAll: () => cookieStore.getAll() } }
  )
}
```

### Tablet-First Responsive (da frontend spec secao 6)

O painel do restaurante tem **prioridade tablet** porque sera usado no balcao, muitas vezes em tablets. Isso significa:
- **Botoes minimo 48px** de altura (touch target adequado)
- **Touch targets generosos** em toda a interface — nada pequeno demais para dedos
- **Breakpoints:**
  - Mobile (< 640px): sidebar drawer, tabelas viram cards empilhados
  - Tablet (640-1023px): sidebar overlay, layout single column
  - Desktop (1024px+): sidebar persistente, layout completo
- **Tela de validacao de cupom:** deve ser especialmente otimizada para tablet (botoes grandes, camera preview grande)
