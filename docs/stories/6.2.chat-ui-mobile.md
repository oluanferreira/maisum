## Story
**As a** client user,
**I want** to chat with restaurants directly,
**so that** I can ask questions or confirm availability.

## Acceptance Criteria
1. Tab "Chat" lista todas as conversas do usuario, ordenadas por ultima mensagem (FR26)
2. Badge de mensagens nao lidas na tab Chat e em cada conversa individualmente (FR26)
3. Tela de conversa com: header (nome + foto do restaurante), mensagens em bolhas (ChatBubble), input de texto + botao enviar (FR26)
4. Mensagens aparecem em tempo real sem refresh — Supabase Realtime subscription ativa enquanto tela aberta (FR26, NFR7)
5. Indicador de "digitando..." (opcional, nice-to-have) (FR26)
6. Scroll automatico para ultima mensagem ao abrir conversa e ao receber nova mensagem (FR26)
7. Iniciar nova conversa a partir da pagina de detalhes do restaurante (botao "Conversar com o restaurante") (FR26)

**Realtime Subscription (from Architecture section 10.2):**
```typescript
// hooks/useChat.ts
const channel = supabase
  .channel(`conversation:${conversationId}`)
  .on('postgres_changes', {
    event: 'INSERT',
    schema: 'public',
    table: 'messages',
    filter: `conversation_id=eq.${conversationId}`,
  }, (payload) => {
    addMessage(payload.new as Message)
  })
  .subscribe()
```

**Zustand Chat Store (from Architecture section 9.3):**
```typescript
// stores/chat.ts
interface ChatState {
  conversations: Conversation[]
  activeConversation: string | null
  messages: Record<string, Message[]>
  unreadCount: number
  subscribe: (conversationId: string) => void
  sendMessage: (conversationId: string, content: string) => Promise<void>
}
```

**useChat Hook Pattern:**
```typescript
// hooks/useChat.ts
function useChat(conversationId: string) {
  const { messages, sendMessage, subscribe } = useChatStore()

  useEffect(() => {
    subscribe(conversationId)
    return () => { /* unsubscribe cleanup */ }
  }, [conversationId])

  return { messages: messages[conversationId] || [], sendMessage }
}
```

**Key decisions:**
- FlashList para lista de mensagens (performance com centenas de mensagens)
- Cursor-based pagination (created_at) para load more — nao OFFSET
- Keyboard avoiding view nativa do Expo (KeyboardAvoidingView)
- Badge count via query separada + Realtime update (nao recalcular client-side)
- Conversa 1:1 por par usuario-restaurante (UNIQUE constraint no banco)
