## Story
**As a** client user,
**I want** to filter and search restaurants,
**so that** I can find exactly what I'm looking for.

## Acceptance Criteria
1. Lista de restaurantes em cards: foto principal, nome, tipo de cozinha, beneficio principal, distancia, nota media (FR10, FR11)
2. Barra de busca textual no topo (busca por nome e tipo) (FR13)
3. Filtros: tipo de cozinha (multi-select), distancia (slider), beneficio disponivel agora (toggle) (FR11)
4. Ordenacao: mais proximo, melhor avaliado, mais recente (FR11)
5. Skeleton loading durante carregamento (NFR1)
6. Estado vazio quando nenhum restaurante encontrado ("Nenhum restaurante encontrado nesta area") (UX)
7. Paginacao infinita (scroll para carregar mais) (NFR6)

**Supabase Query Pattern para filtros:**
```typescript
// services/restaurants.ts
export async function searchRestaurants(params: {
  query?: string
  cuisineTypes?: string[]
  maxDistance?: number
  availableNow?: boolean
  sortBy?: 'nearest' | 'best_rated' | 'newest'
  lat: number
  lng: number
  page?: number
  pageSize?: number
}) {
  let query = supabase
    .from('restaurants')
    .select(`
      *,
      benefits!inner (name, category, is_active),
      benefit_rules (available_days, available_hours_start, available_hours_end, is_active),
      reviews (rating)
    `)
    .eq('is_active', true)

  if (params.query) {
    query = query.or(`name.ilike.%${params.query}%,cuisine_type.ilike.%${params.query}%`)
  }

  if (params.cuisineTypes?.length) {
    query = query.in('cuisine_type', params.cuisineTypes)
  }

  // Pagination
  const page = params.page ?? 0
  const pageSize = params.pageSize ?? 10
  query = query.range(page * pageSize, (page + 1) * pageSize - 1)

  return query
}
```
