## Story
**As a** client user,
**I want** to earn an extra coupon by sharing on social media,
**so that** I'm rewarded for promoting the restaurant and +um.

## Acceptance Criteria
1. Tabela `social_proofs` criada: id, user_id, restaurant_id, coupon_id, proof_type (screenshot | link), proof_url, status (pending | approved | rejected), reviewed_by, created_at (FR36-FR40)
2. Apos usar cupom, botao "Compartilhar e ganhar +1" na tela de sucesso (FR36)
3. Share sheet com texto pre-preenchido mencionando @maisumapp e @restaurante (FR37)
4. Upload de screenshot ou colar link como prova (FR38)
5. Prova enviada fica com status "pending" (FR38)
6. Restaurante e admin +um podem ver provas pendentes e aprovar/rejeitar (FR39, FR46, FR54)
7. Ao aprovar, usuario recebe 1 cupom extra (respeitando limite de 10/mes) (FR40)
8. Notificacao push ao usuario quando prova aprovada/rejeitada (FR55)

## Dev Notes

**Social Proofs Table DDL (from Architecture doc section 5.1):**
```sql
CREATE TYPE social_proof_status AS ENUM ('pending', 'approved', 'rejected');
CREATE TYPE social_proof_type AS ENUM ('screenshot', 'link');

CREATE TABLE social_proofs (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  restaurant_id UUID NOT NULL REFERENCES restaurants(id),
  coupon_id UUID REFERENCES coupons(id),
  proof_type social_proof_type NOT NULL,
  proof_url TEXT NOT NULL,
  status social_proof_status DEFAULT 'pending',
  reviewed_by UUID REFERENCES profiles(id),
  created_at TIMESTAMPTZ DEFAULT now()
);

CREATE INDEX idx_social_proofs_pending ON social_proofs(status) WHERE status = 'pending';
CREATE INDEX idx_social_proofs_restaurant ON social_proofs(restaurant_id, status);
```

**RLS Policies (from Architecture doc):**
```sql
CREATE POLICY "Users view own proofs" ON social_proofs FOR SELECT USING (user_id = auth.uid());
CREATE POLICY "Users can submit proofs" ON social_proofs FOR INSERT WITH CHECK (user_id = auth.uid());
CREATE POLICY "Restaurant admin views their proofs" ON social_proofs FOR SELECT USING (restaurant_id = get_my_restaurant_id());
CREATE POLICY "Restaurant admin can review proofs" ON social_proofs FOR UPDATE USING (restaurant_id = get_my_restaurant_id());
CREATE POLICY "Super admin full access" ON social_proofs FOR ALL USING (get_user_role() = 'super_admin');
```
